# [문제 1] 회원가입 데이터 정제 및 아이디 자동 생성 프로그램

활용 범위: String (length, trim, replace, toLowerCase, padStart), Array (push, join, map)

개요: 사용자가 입력한 회원 정보의 공백과 대소문자를 정제하고, 이메일 주소 및 가입 순서를 기반으로 고유한 회원 아이디를 자동 생성하는 프로그램입니다.

## 1. 필수 기능 요구사항

기능 1: 입력 데이터 정제하기

사용자가 입력한 이름과 이메일의 앞뒤 공백을 trim()을 사용해 제거하라.

이메일 주소는 대소문자 구분을 없애기 위해 전체 toLowerCase()를 적용하라.

## 기능 2: 예외 처리 조건 체크

이름의 글자 수(length)가 2글자 미만인 경우 "이름을 정확히 입력하세요"라는 경고창(alert)을 띄우고 처리를 중단하라.

## 기능 3: 고유 회원 아이디 생성하기

이메일의 @ 앞부분 문자열만 추출하거나, 이메일 문자열 내의 특정 특수문자를 replace()로 제거하여 베이스 아이디를 추출하라.

현재 가입한 회원 순서(배열의 length 활용)를 기반으로, 회원 번호가 항상 4자리가 되도록 모자란 앞부분을 0으로 채우는 padStart(4, "0") 함수를 사용하라. (예: 3번째 회원이면 0003)

두 값을 결합하여 USER\_베이스아이디\_0003 형태의 최종 회원 아이디를 완성하고 회원 배열(userList)에 객체로 추가하라.

## 기능 4: 가입 회원 목록 화면 출력

map()과 join()을 활용하여 현재까지 가입된 회원들의 아이디와 정제된 이메일을 테이블(<tbody>) 영역에 동적으로 출력하라.

# [문제 2] 도서관 도서 대여 기록 및 반납 상태 계산기

활용 범위: Date (new Date, get 함수들, getTime), Number (parseInt), Array (push, map, join, reduce)

개요: 도서 대여일과 반납 예정일을 관리하고, 오늘 날짜를 기준으로 반납까지 남은 일수 또는 연체 일수를 계산해 주는 프로그램입니다.

## 1. 필수 기능 요구사항

기능 1: 기본 반납 예정일 설정 (대여일 기준 +7일 계산)

사용자가 대여일을 지정하지 않으면 오늘 날짜(new Date())를 기본값으로 사용하라.

대여일 객체의 getTime() 값에 7일 치의 밀리초(7 _ 24 _ 60 _ 60 _ 1000)를 더해 반납 예정일 Date 객체를 생성하라.

생성된 Date 객체에서 getFullYear(), getMonth() + 1, getDate()를 추출하여 YYYY-MM-DD 포맷 문자열로 변환한 뒤 화면에 기본 표시하라.

## 기능 2: 대여 도서 추가하기

입력된 도서명, 대여일, 반납 예정일을 객체로 묶어 rentalList 배열에 push() 하라.

## 기능 3: 연체 여부 및 남은 일수 계산하기 (Date 활용)

오늘 날짜와 반납 예정일의 밀리초 차이를 구한 뒤, 하루 치 밀리초(24 _ 60 _ 60 \* 1000)로 나누어 남은 일수를 계산하라.

예정일이 오늘보다 지났다면 "n일 연체", 아직 남았다면 "n일 남음"을 계산하여 테이블에 출력하라.

## 기능 4: 총 대여 도서 수량 합산

reduce()를 사용하여 현재 대여 중인 총 도서 권수를 계산하여 화면 하단에 표시하라.

# [문제 3] 쇼핑몰 상품 가격 필터 및 영수증 발행 프로그램

활용 범위: Number (toFixed, parseInt), Array (filter, sort, reduce, map, join)

개요: 상품 목록 데이터를 가격순으로 정렬하거나 특정 가격대만 필터링하고, 장바구니에 담긴 상품의 총 금액 및 평균 가격을 계산하는 프로그램입니다.

## 1. 필수 기능 요구사항

기능 1: 상품 가격 정렬하기 (오름차순/내림차순)

sort() 함수와 비교 알고리즘 (a, b) => a - b 또는 b - a 를 구현하여 상품 가격을 기준으로 정렬된 새 배열을 만들고 화면에 렌더링하라.

## 기능 2: 예산 맞춤 상품 필터링

사용자가 <input> 태그에 입력한 최고 금액 숫자를 parseInt()로 가져와 조건문을 작성하라.

filter()를 사용하여 입력된 예산 이하의 가격을 가진 상품들만 골라내어 화면에 출력하라.

## 기능 3: 장바구니 총액 및 평균가 계산

reduce()를 활용하여 장바구니 배열안에 담긴 모든 상품의 누적 합산 금액을 구하라.

추가로 장바구니 상품들의 평균 가격을 구한 뒤 소수점 아래 둘째 자리까지만 표시되도록 toFixed(2)를 적용하여 출력하라.

# [문제 4] 연락처 패턴 검색 및 이메일 도메인 분류 프로그램

활용 범위: String (startsWith, endsWith, slice, split), Array (filter, map, join)

개요: 저장된 연락처 목록에서 사용자가 입력한 번호의 시작/끝 조건을 검색하고, 유효하지 않은 형식의 전화번호를 통일된 포맷으로 변환해 주는 프로그램입니다.

## 1. 필수 기능 요구사항

기능 1: 전화번호 포맷팅 정제

사용자가 하이픈(-) 없이 입력한 연락처 문자열을 slice()를 사용하여 010-XXXX-XXXX 형태로 강제 변환하여 저장하라.

## 기능 2: 특정 통신사 식별 및 번호 검색

startsWith('010') 함수를 사용하여 올바른 휴대폰 번호 형식으로 시작하는지 체크하고, 아니면 경고 문구를 띄워라.

사용자가 뒷자리 4자리를 입력하고 검색했을 때 endsWith(검색어)를 사용하여 해당 번호로 끝나는 연락처 목록만 filter()로 추출하라.

## 기능 3: 이메일 도메인 분류 (split 활용)

연락처에 등록된 이메일 주소를 split('@')을 이용해 분리하고, 뒤쪽 도메인(naver.com, gmail.com 등)만 추출하여 화면에 표시하라.

## 기능 4: 연락처 전체 보기

map()과 join()을 활용하여 연락처 목록을 이름, 정제된 전화번호 형태의 <li> 태그 리스트로 출력하라.

# [문제 5] 은행 계좌 입출금 내역서 및 거래 제한시간 체크 프로그램

활용 범위: Date (getHours, getMinutes), Number (isNaN, isFinite, parseInt), Array (filter, reduce, push)

개요: 은행 계좌의 거래 내역을 기록하되, 야간 점검 시간대의 거래를 제한하고 입력된 거래 금액의 유효성을 검사하는 프로그램입니다.

## 1. 필수 기능 요구사항

기능 1: 거래 금액 유효성 검사 (전역 함수 활용)

입금액 또는 출금액 입력 칸의 값을 가져와 isNaN()을 통해 문자가 섞여 있는지 검사하고, 숫자가 아닐 경우 "올바른 금액을 입력하세요"라고 예외 처리하라.

값이 정상적인 범위의 숫자인지 isFinite()로 더블 체크하라.

## 기능 2: 은행 야간 점검 시간대 거래 제한 (Date 활용)

거래 버튼을 누른 순간의 new Date()财富 객체를 생성하라.

getHours()와 getMinutes()를 사용하여 현재 시간이 밤 23시 30분부터 23시 40분 사이(은행 점검 시간)에 해당하는지 비교 연산자로 확인하고, 이 시간대일 경우 거래가 불가하다는 안내를 띄우고 배열 추가를 중단하라.

## 기능 3: 입출금 유형별 필터링

filter()를 사용하여 거래 내역 배열(historyList)에서 "입금" 내역만 따로 보거나, "출금" 내역만 필터링하여 볼 수 있는 기능을 구현하라.

## 기능 4: 현재 잔액 계산하기

reduce()를 활용하여 "입금" 금액은 더하고, "출금" 금액은 빼는 누적 연산을 수행하여 현재 통장의 최종 잔액을 계산하고 화면에 출력하라.
